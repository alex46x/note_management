# 📝 Premium Notes Management App

A production-ready, beautiful, and fully functional Notes Management application built in Flutter with Cloud Firestore. The project features clean architectural separation (model, views, widgets, and services) and adheres to premium modern Material 3 and custom iOS design directives.

---

## ✨ Features

- **⚡ Real-Time Syncing:** Notes are fetched from Cloud Firestore in real time using responsive streams (`StreamBuilder`).
- **🗂️ Tag Folder Grid View (iOS & Keep Style):** Group notes dynamically by tags into premium folder cards (e.g., *Starters*, *Mains*). Shows note counts and lists of note titles. Click folders to filter, or tap note titles to open details instantly.
- **🏷️ Google Keep-Style Tags:** Add comma-separated tags (e.g. *work, personal*) while writing notes. Renders beautiful live tag chips preview instantly.
- **🖤 Premium Obsidian Dark Mode:** Luxurious OLED-grade dark mode using obsidian black (`#090A0F`) and deep charcoal (`#16171F`) accents, with soft pastel dark card variants.
- **✍️ Plus Jakarta Sans Typography:** Fully integrated geometric font pairing globally using `google_fonts` to provide a clean, high-end Apple-like look.
- **🚨 iOS Delete Confirmation Dialog:** Sleek, custom-designed confirmation popup dialog for swiping and card deletes, matching native iOS specifications.
- **🔄 Swipe-to-Delete with SnackBar Undo:** Dismiss cards instantly with swipe gestures, and restore them using the floating snackbar **UNDO** action.
- **🛡️ Unsaved Changes Guard:** Prevents accidental data loss by prompting a confirmation dialog when exiting with unsaved edits.
- **🚀 Safety Fallback Launch:** Safe initial startup logic showing setup guides if Firebase options are not configured.

## 🔐 Security Notice & GitHub Public Safety

This repository is configured to be public-friendly. The following sensitive configuration files are automatically ignored in the `.gitignore` to prevent database abuse or API keys leakage:

- `lib/firebase_options.dart` (Your Firebase API keys & Project IDs)
- `android/app/google-services.json` (Google Play Services configuration)
- `ios/Runner/GoogleService-Info.plist` (iOS App IDs & database endpoint keys)
- Native keystores (`*.keystore`, `*.jks`), local signing properties, and `.env` config environments.

To configure and run this project on your machine, follow the Firebase setup steps below.

---

## 🔥 Firebase Setup Steps

To connect the application to your own Firebase project:

1. **Install Firebase CLI:**
   Ensure Node.js is installed, then install the CLI globally:
   ```bash
   npm install -g firebase-tools
   ```

2. **Log in to Firebase:**
   ```bash
   firebase login
   ```

3. **Activate the FlutterFire CLI:**
   ```bash
   dart pub global activate flutterfire_cli
   ```

4. **Configure your Firebase Project:**
   Run the following command from the root directory of this project:
   ```bash
   dart pub global run flutterfire_cli:flutterfire configure
   ```
   Select your existing Firebase project or create a new one. This command will automatically generate `lib/firebase_options.dart` with your unique configuration values and register native apps for Android, iOS, Web, macOS, etc.

5. **Enable Cloud Firestore:**
   - Go to your [Firebase Console](https://console.firebase.google.com/).
   - Click on **Firestore Database** in the left sidebar.
   - Click **Create Database**.
   - Start in **Production Mode** (and publish rules allowing read/write access to the `notes` collection) or **Test Mode** for testing.
